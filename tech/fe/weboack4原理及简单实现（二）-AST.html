webpack使用各种loader来编译模块，那么具体的编译是怎么实现的呢。这时候就轮到主角AST--抽象语法树登场了。

在计算机科学中，抽象语法树（abstract syntax tree 或者缩写为 AST），或者语法树（syntax tree），是源代码的抽象语法结构的树状表现形式，这里特指编程语言的源代码。树上的每个节点都表示源代码中的一种结构。之所以说语法是「抽象」的，是因为这里的语法并不会表示出真实语法中出现的每个细节。

这棵树定义了代码本身，通过操作这颗树，可以精准的定位到赋值语句、声明语句和运算语句。对树的节点遍历修改属性，可以帮助我们对代码做各种修改。

AST的使用场景非常广泛，包括代码语法的检查、代码风格的检查、代码的格式化、代码的高亮、代码错误提示、代码自动补全等等。

将javascript源码转换成抽象语法树需要借助引擎，现在常用的引擎有

  • esprima
  • acron (webpack使用的)
  • Traceur
  • UglifyJS2
  • shift

这是在线的AST转换器：AST转换器。我们使用babylon引擎将代码 const a = 1 转化成AST后的格式如下所示：

各字段的说明如下:

type:描述该语句的类型 --变量声明语句
kind：变量声明的关键字 -- var
declaration: 声明的内容数组，里面的每一项也是一个对象

	id: 描述变量名称的对象

		name: 是变量的名字
    init: 初始化变量值得对象

		value: 值 "is tree" 不带引号
		row: "\"is tree"\" 带引号

AST的使用步骤可以归纳为3步：

  1. 通过引擎将代码转换陈AST
  2. 遍历和更新AST
  3. 将更新之后的AST再转换成代码

下面我们以acron为例，简单的演示一下AST的应用方式。我们希望把 const a = 1 这句代码编译成 var a = 1

# 一、通过引擎将代码转换成AST

我们新建一个工程，安装依赖 npm i acorn estraverse escodegen
写好源码

const acorn = require('acorn')
const estraverse = require('estraverse')
const escodegen = require('escodegen')

let source = 'const a = 1'

通过引擎将代码转换成AST需要调用babylon中的parse方法

const acorn = require('acorn')
const estraverse = require('estraverse')
const escodegen = require('escodegen')

let source = 'const a = 1'

const ast = acorn.parse(source, {
	sourceType: "script",
  locations: true
})

console.log(ast)

打印生成的ast，和我们上面通过AST转换器转换生成的结果是一样的。这样我们就完成了第一步

# 二、遍历和更新AST

上述代码中引入的traverse就是用来遍历AST的。

const acorn = require('acorn')
const estraverse = require('estraverse')
const escodegen = require('escodegen')

let source = 'const a = 1'

const ast = acorn.parse(source, {
	sourceType: "script",
  locations: true
})

console.log(ast)

estraverse.traverse(ast, {
	enter(node) {
    	// 找到kind为const的node
		if (node.kind === 'const') {
        	// 将kind修改为var
			node.kind = 'var'
		}
	}
})

# 三、将更新之后的AST再转换成代码

AST修改之后，我们需求将它再转换成代码，这一步使用escodegen模块

const acorn = require('acorn')
const estraverse = require('estraverse')
const escodegen = require('escodegen')

let source = 'const a = 1'

const ast = acorn.parse(source, {
	sourceType: "script",
  locations: true
})

console.log(ast)

estraverse.traverse(ast, {
	enter(node) {
    	// 找到kind为const的node
		if (node.kind === 'const') {
        	// 将kind修改为var
			node.kind = 'var'
		}
	}
})

// 将ast转化成代码
let code = escodegen.generate(ast)

console.log(code)   // var a = 1

经过上面三个步骤，我们成功的将代码进行了修改。webpack中对模块的编译也和上述过程类似，只是代码逻辑更复杂，有兴趣的同学可以对AST深入研究。
